Superadmin Role Guide

Full system access. You are the system owner — responsible for user management, project stage control, system-wide visibility, and all reporting.

Section 1

Introduction

The Superadmin role has unrestricted access to every part of WoodWings PMS. With that comes responsibility for keeping the system healthy and the team unblocked.

Your Role: Superadmin

As Superadmin, you own the following functions that no other role can perform:

  • Setting the outcome of "Call the Client" tasks (Approved / Needs Review / Refused)
  • Manually advancing or managing project stages
  • Creating, editing, and deactivating user accounts
  • Managing the worker roster (for timesheets)
  • Viewing the full system KPI overview and SED performance chart
  • Downloading all Excel reports across Sales, Accountant, Material, and Calendar categories
  • Viewing all team members' active tasks on the Team Activity page
  • Posting system-wide announcements
  • Deleting timesheet entries
  • Viewing the superadmin timesheets page with full filter, edit, and export capability

How to Log In

Navigate to /login, enter your credentials, and you are redirected to /dashboard/superadmin.

To log out, use the menu at the bottom-left of the sidebar and select Log Out.

Keep credentials secureYour account can affect all users and all data. Never share your password. If you suspect a breach, change your password immediately and contact IT.

Section 3

Task Management

As Superadmin you can see and action all tasks across all roles. Your most important unique task is the "Call the Client" decision.

Status Flow

Locked → To Do → In Progress → Pending Approval → Completed

Call the Client — Your Exclusive Task

When all Phase 1 gate checks pass (Quotation, Sample, and/or Concept Design all approved), the Call the Client task unlocks and is assigned to you. This is the single most important action in Phase 1 — your decision determines the project's path.

1
Find the taskIt appears in your My Tasks tab with a distinct 3-button panel.
2
Call the clientMake the call outside the system, then record the outcome.
3
Select the outcome:
  • Approved — client confirmed. Project advances to Phase 2 (Open stage). Notifications sent to Accountant, Fabrication, and Manager.
  • Needs Review — client wants changes. All Phase 1 action tasks reset to To Do for SED to rework.
  • Refused — client declined. Project is marked Not-Approved.
This cannot be undone lightlySetting a project to Refused or triggering a Needs Review resets significant work. Confirm with the SED before selecting Refused.

Approving and Rejecting Tasks

You have the same approval capability as the Manager. Tasks in Pending Approval appear in your My Tasks view with Approve / Reject buttons.

Section 4

System Administration

Managing Users

Go to Users tab on the superadmin dashboard.

ActionHow
Add a new userClick Add User → fill in name, email, role, and set a password → Save
Edit a userClick Edit on the user row → update fields → Save
Deactivate a userToggle the Active switch on the user row. Deactivated users cannot log in.
Reset a user's roleEdit the user → change the Role dropdown → Save
Deactivating vs deletingAlways deactivate rather than delete a user. Deactivated users' historical data remains intact. Deletion removes them permanently.

Managing Workers (Timesheet Roster)

Go to Workers in the sidebar. Workers are daily labourers who appear in the timesheet entry form — separate from system users.

ActionHow
Add a workerClick Add Worker → enter name, full name, nickname, role, check Active → Save
Edit a workerClick Edit on the worker row → update fields → Save
Deactivate a workerClick the Active/Inactive badge on the row to toggle. Inactive workers do not appear in the timesheet dropdown.
Delete a workerClick Delete on the row. Confirmation required. Only delete if the worker has no historical timesheet data.

Posting Announcements

Go to Announcements tab. Click New Announcement. Fill in the title, message body, select who can see it (Visible To), set an optional expiry date, and optionally pin it to the top of the home page for all users.

Team Activity Page

Go to Team Activity in the sidebar. This page shows every active team member with their current active tasks, grouped by person. Click a person card to expand their task list. Tasks refresh every 60 seconds automatically.

Accountant Email Setting

The accountant email address (used for payment notifications) can be set from the Users tab. At the bottom of that page is an Email Settings card — click the edit icon, update the address, and save. The default address is aisolutions963@gmail.com.

Notifications Page

Click the bell icon in the left sidebar to open the Notifications page (/dashboard/notifications). Shows your full notification history with all/unread filter. Note: live-computed alerts in the bell badge (pending approvals, stale projects) are not stored here — only system-generated notification records appear.

Section 6

Project Workflow Control

As Superadmin you are the gatekeeper for the most critical project transitions.

Phase 1 → Phase 2 (Preparing → Open)

This transition happens when you set the Call the Client outcome to Approved. The system then:

  • Advances the project to Open stage
  • Sends notifications to Accountant, Fabrication, and Manager
  • Unlocks F4 (advance payment) and F5 (quotation items) tasks

3-Day Inactivity Rule

The system automatically monitors project activity. If a project in Preparing stage has no task set to In Progress for 3 consecutive days, an escalation is triggered:

  • Manager receives a notification
  • Project may be marked Not-Approved automatically

You can review these escalations in the Activity tab and take action if the inactivity was justified (e.g., client was travelling).

Maintenance Period

After a project closes, it enters a 1-year maintenance period:

  • Closed & Valid Maintenance — within the 1-year warranty window
  • Closed & Warranty Done — after 1 year from close date

Monitor these in the Warranty tab.

Section 7

Common Scenarios

Scenario 1 — A new team member joins. How do I set them up?

1
Go to Users tab → Add User.
2
Enter their name, email address, and select their role (sed / manager / fabrication / installation).
3
Set a temporary password and ensure Active is checked.
4
Save. The user can now log in and will be routed to their role-specific dashboard.

Scenario 2 — The "Call the Client" task is in my queue. What do I do?

1
Check with the SED owner for the latest client status before making the call.
2
Call the client personally or confirm the SED has done so.
3
Open the task in My Tasks → select the outcome: Approved, Needs Review, or Refused.
4
If Approved — the project moves to Open and the whole team is notified.

Scenario 3 — I want to see how many hours a specific worker logged this month

1
Go to Timesheets in the sidebar.
2
Set the date range to the current month (From: first of month, To: today).
3
Select the worker from the Worker filter dropdown.
4
The summary chips show total regular, overtime, and grand total hours. Click Export Excel for a detailed breakdown.

Scenario 4 — A SED is not performing. How do I check their pipeline?

On the Overview tab, look at the SED Performance Chart. Switch to the individual SED tab to see their project counts by stage. For a full list, go to All Projects and filter by the SED's name in the Sales Owner column. You can also download the SED Projects Status report from the Reports section.

Scenario 5 — An employee has left. How do I remove their access?

1
Go to Users tab. Find the employee.
2
Toggle the Active switch to Inactive. Their session is invalidated immediately.
3
Do not delete the user — their historical data and task records are preserved.

Troubleshooting Guide

ProblemSolution
A user says they cannot log inCheck their account in Users tab — ensure Active is toggled on and their email is correct.
A project is stuck and no tasks are unlockingOpen Phase Gates view — identify the blocking gate. If it is Pending Approval, find and approve or reject it.
Call the Client task does not appear in My TasksThe Phase 1 gate checks may not all be complete yet. Check the project — at least one of Quotation/Sample/Design must be approved.
A report download returns an empty fileNo data exists for the selected date range. Widen the date filter.
A worker does not appear in the manager's timesheet dropdownGo to Workers page — check the worker is Active.
An announcement is not visible to a specific roleEdit the announcement and check the Visible To field — ensure the correct role is selected.
The 3-day rule triggered incorrectly on a projectThe escalation is automatic. Review the Activity log for the project and contact the SED owner. You can re-open or reassign if justified.
