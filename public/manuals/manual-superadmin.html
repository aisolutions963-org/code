1. What Is This System?

WoodWings PMS is the company's project management system. It tracks every project from client enquiry through design, quotation, fabrication, installation, and handover.

As the Superadmin (Kanaan), you have full access to everything in the system. Your unique responsibilities include:

  • Making the Call the Client decision (Approved / Needs Review / Rejected) for every project
  • Advancing project stages manually when needed
  • Reopening closed or rejected projects
  • Leaving Superadmin Notes (amber banners) on any task for other users
  • Managing user accounts and worker records
  • Reviewing all reports (Sales, Accountant, Material, Calendar, Clients)
  • Managing the warranty/maintenance lifecycle (manually expiring maintenance if needed)
  • Posting company announcements
  • Deleting projects (permanent — use with care)

2. How to Log In

  1. Open your web browser and navigate to the WoodWings app URL.
  2. Enter your email address and password.
  3. Click Sign In.
  4. You land on the Superadmin Dashboard — the most complete view in the system.

5. How to Work a Task

As superadmin you can edit any field on any task. The standard workflow for your own tasks:

  1. Go to your My Tasks section on the Overview page, or navigate to the Tasks page.
  2. Click the task to open it.
  3. Change status to In Progress, fill required fields.
  4. Mark as Completed when done.

For approval tasks from other team members, review and click Approve or Reject (with a comment on rejection).

6. Your Special Powers

Call the Client — The Big Decision

This is the most important action in the system. After SED prepares a project (quotation, samples, site visit), the Call the Client task unlocks — and only you can decide its outcome.

  1. The task appears in your My Tasks section (or you can find it in the Tasks page).
  2. Open it. You will see three large buttons:
ButtonLabelWhat Happens
GreenApproved — "Project advances to Phase 2"Project moves to Open stage. Per-item tasks are generated for all F5 quotation items. Production begins.
AmberNeeds Review — "Action tasks reset to To Do"SED's action tasks are all reset to To Do. They must rework and resubmit.
RedRejected — "Project marked Not-Approved"Project is marked Not-Approved immediately. No further tasks are created.
The Rejected button has a confirmation step. You will be asked to confirm before the system records the rejection. Read it carefully before clicking yes.
📸

[SCREENSHOT: Call the Client panel — three large buttons (green Approved, amber Needs Review, red Rejected) with confirmation step visible]

Advancing Project Stages

If a project needs to be manually advanced (for example, all tasks are complete but the stage hasn't updated), you can do this from the Projects page.

  1. Go to the Projects page from the sidebar.
  2. Find the project row.
  3. Click Advance →.
  4. Confirm the action. The project moves to the next stage.
Only advance a stage manually if you are certain all required tasks are complete. The system normally advances stages automatically when tasks are done.

Reopening a Project

If a project was incorrectly rejected or closed, you can reopen it:

  1. Go to the Projects page.
  2. Find the project row (it will show as Not-Approved or Closed).
  3. Click ↩ Reopen.
  4. The project is restored to its previous active stage.

Superadmin Note (Amber Banner)

You can leave a special note on any task that will appear as an amber (yellow/orange) banner at the top of that task for all other users. Other users cannot edit or remove it — they can only read it.

Use this to communicate specific client instructions, warnings, or anything that a team member must know before completing that task.

  1. Open any task.
  2. Find the Superadmin Note field.
  3. Type your message and save.
  4. All users who open that task will see the amber banner immediately.
The amber banner is visible to all roles. If you leave a note, every team member who opens the task will see it — SED, Fabrication, Installation, and Manager.

Manually Expiring Maintenance

Projects that are Closed & Valid Maintenance have a 1-year warranty period. The system expires them automatically when 12 months pass. However, you can manually expire a maintenance period if needed:

  1. Go to the Warranty page from the sidebar.
  2. Find the project.
  3. Click Expire Maintenance (or the equivalent button for manual expiry).
  4. Confirm. The project stage moves to Closed & Warranty Done.

7. Admin Tools

User Management

Go to the Users page from the sidebar. From here you can:

  • Create a new user — click + Add User, enter email, name, and assign a role (sed, manager, fabrication, installation, superadmin).
  • Reset a password — find the user row and click Reset Password. The system generates a new temporary password.
  • Change a user's role — edit the user record and select a new role.
  • Deactivate a user — toggle the user off to prevent login without deleting their history.
📸

[SCREENSHOT: Users page — user list table with Add User button and action buttons per row]

Worker Management

Workers are the installation team members who appear in gate passes and timesheets. To manage them, go to the appropriate section (part of the Users/Admin area). You can add workers, edit their details (name, phone), and deactivate workers who have left.

Announcements

Post a message to all users from the Announcements page:

  1. Go to Announcements in the sidebar.
  2. Click + New Announcement.
  3. Write your message. You can mark it as urgent (shows as a red banner) or standard.
  4. Click Post. All users see it the next time they open their dashboard.

Deleting a Project

From the Projects page, each row has a Delete button.

⚠ Permanent and irreversible. Deleting a project removes it and all its associated tasks, materials, payments, and history. There is no undo. Only delete test projects or clearly duplicate entries. If in doubt, use the ↩ Reopen or ✕ Not-Approved action instead.

9. Frequently Asked Questions

How do I advance a project from Preparing to Open?

Use the Call the Client task for that project. Open it and click Approved. This is the correct way. Only use the manual Advance → button on the Projects page as a last resort (e.g., when the Call the Client task is missing or the project is in an unusual state).

A SED user accidentally clicked "Reject Project" — how do I undo it?

Go to the Projects page. Find the project (now showing as Not-Approved). Click the ↩ Reopen button. The project will be restored to its previous active stage and all tasks will be accessible again.

What happens when I leave a Superadmin Note on a task?

The note appears as an amber (yellow/orange) banner at the top of the task for every user who opens it — SED, Fabrication, Installation, and Manager. They can read it but cannot edit or remove it. Only you can change or clear it. Use it for important client-specific instructions or warnings.

When does Phase 4 (Installation Completed) trigger automatically?

Phase 4 triggers automatically when all per-item tasks across all items in the project are marked as Completed. It does not trigger based on a single task name — every item's task chain must be fully complete. If you believe Phase 4 should have triggered but hasn't, check the Tasks page for that project and look for any remaining open tasks.

How do I manually expire a maintenance/warranty period?

Go to the Warranty page from the sidebar. Find the project. Use the manual expire action on that row. This moves the project from "Closed & Valid Maintenance" to "Closed & Warranty Done". Only do this if the actual 1-year period has passed or a client has waived the remaining warranty in writing.

Why did a project NOT get tasks when I clicked Approved?

Tasks are generated per F5 quotation item. If the SED team has not submitted the F5 form (with quotation line items) yet, there are no items to generate tasks from. Ask the SED team to submit the F5 form on the project page first, then you can set the Call the Client outcome.

Can I post an announcement only for one role?

Currently all announcements are company-wide — all roles see them. If you need to communicate something to a specific team, use the Superadmin Note feature on a relevant task for that team, or contact them directly.

The weekly cron reminder isn't sending — what do I do?

The weekly reminders are sent via GitHub Actions (not the app server itself). If reminders stop coming, check with the developer that the GitHub Actions cron workflow is still active and the environment secrets (auth tokens) haven't expired.
