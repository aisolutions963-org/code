Superadmin Manual

Full system access — projects, people, payments, and everything in between.

Dashboard Overview

The dashboard is your command centre. It loads when you log in and shows the current state of the entire business.

KPI Cards

Eight cards at the top show live project counts by stage. Click any card to open the filtered project list. Click the Excel icon on a card to download that subset.

CardWhat it counts
Total ProjectsEvery project in the system
PreparingProjects in early scoping / quotation stage
OpenApproved projects — items being designed
Not ApprovedRejected or on-hold projects
FinishedFully closed projects
Active WarrantyClosed and inside the 1-year warranty window
UnpaidFinished but final payment not yet received
Warranty ExpiredWarranty period ended

SED Performance Chart

A bar chart below the KPI cards shows each SED's project count broken down by stage. Click a SED's name to drill into their individual projects. The commission table below the chart estimates each SED's commission at 1.5% of amounts collected.

Reports Section

Scroll further down on the dashboard to reach the Reports section — five report categories with quarter/year filters and Excel download buttons.

Home Page

The home page (house icon in the sidebar) shows announcements and the system calendar.

  • Pinned announcements appear at the top with an amber highlight.
  • Expired announcements are automatically hidden.
  • The embedded calendar shows upcoming project events.

Pipeline

The pipeline (sidebar → Pipeline) is a kanban board showing all active projects grouped by stage.

1Open the pipeline.

Five columns: Preparing, Open, Production, Done, Warranty. Each card shows the project name, client, location, and a payment progress bar.

2Search for a project.

Type in the search bar at the top. Results filter live across all columns by project name, client, or ID.

3Click a project card.

A side drawer shows client info, stage, payment progress, and description. Click Open Project Page to view the full project.

Only superadmin and manager can see the pipeline. Projects in Closed and active warranty appear in the Warranty column.

Creating a Project

1Click "+ New Project" on the dashboard.
2Fill in the project details.
Project NameRequired. Must be unique.
Client NameFull client name.
Client PhoneContact number for the site.
SED OwnerThe assigned sales rep.
Quotation NumberReference from the quotation.
Emirate / LocationWhere the project is.
Payment ModeHow payment is structured.
DescriptionShort scope summary.
3Submit.

The project is created in Preparing stage and the first set of tasks is auto-generated.

Duplicate project names are blocked. If you see an error, search the project list first.

Phase Gates

Phase Gates (sidebar → Phase Gates) lists all active projects and lets you advance them to the next stage once their tasks are done.

1Find the project you want to advance.

Each project card shows how many tasks are complete vs total. Expand the card to see the blocking tasks by name and status.

2Click "Advance →".

If all required tasks are Completed, the project moves to the next stage and new tasks are generated. If any task is not done, you'll see a list of what's still blocking.

Stage flow: Preparing → Open → Production → Closed. Once closed and a final payment is recorded, the stage becomes Closed and active warranty.

Project Detail Page

Click any project name to open its detail page. It has two tabs.

Tasks & Items Tab

Shows all project tasks in a board view, plus production items. Expand any task card to update its status, change fields, and add notes.

Project Report Tab

A full financial and operational summary:

  • Project Overview — client, phone, SED, quotation number, location, and description.
  • Payments — contract value, total collected, remaining balance, progress bar, and a full payment history table.
  • Labour Hours — total regular and overtime hours logged against the project, with estimated labour cost.

Client Requests

Client Requests (sidebar → Client Requests) shows follow-on work tied to existing projects. Three types:

TypeWhen to use
TradeClient wants extra items beyond the original project scope
MaintenanceRepair or warranty work on a closed project
VarianceScope change that needs separate tracking

Each request shows the linked parent project, a task progress ring, and the request type. Click a row to open the detail view. SED creates requests — you review and track them here.

My Tasks

Your personal task queue. Three types of items land here.

Call-Client Decisions

When a SED marks a "Call the client" task done, it appears here for your decision. Review the project and client info, then choose:

  • Approve — the project proceeds to the next phase.
  • Needs Review — flag it for further discussion without approving or rejecting.
  • Refuse — the project is marked Not Approved.

Follow-Up Decisions

Projects with no activity for 3+ days generate a follow-up task for you. Choose:

  • Reject Project — closes it as Not Approved.
  • Ask SED to Follow Up — sends a notification to the assigned SED.
  • Ask Manager to Follow Up — notifies the manager.

Pending Approvals

Tasks that other roles submitted for your review. Expand the task card, review the details, then update the status to Completed or leave a rejection note.

Adding an Admin Note to Any Task

1Open and expand any task card.
2Find the amber "Admin Follow-up Note" field and type your note.

It saves automatically. All team members assigned to that task will see it as a pinned amber banner — they can read it but cannot edit it.

Team Activity

Dashboard → Activity shows what the whole team is working on right now.

View by Task

A flat table of all incomplete tasks across the system. Filter by department using the tabs (All / SED / Fabrication / Installation / Management). Toggle the priority flag on any task to mark it urgent — it shows a 🚩 flag for that team member.

View by Person

Expandable cards per team member, showing each person's active tasks with project reference, task name, and status.

Payment Tracker

Sidebar → Payments (or Finance → Payments). See and record all financial activity.

Portfolio Summary

Total contract value, total collected, remaining balance, and overall collection rate across all projects.

Recording a Payment

1Find the project in the payments table and expand it.
2Click "+ Add Payment" and fill in the form.
Payment TypeAdvance / Delivery / Material / Final / Progressive
StatusReceived / Pending / Overdue / Cancelled
Amount (AED)Payment amount.
MethodBank Transfer / Cash / Cheque
DateWhen paid or due.
Reference No.Invoice or receipt number.
Payer TypeBroker / Contractor / End User / Designer
Commission %Broker payments only.
3Submit.

If it's a Final payment, the project automatically closes and the warranty period starts.

Editing a Payment

Click the pencil icon on any payment row. An inline form opens — update fields and save. Only you can edit payments.

Voiding a Payment

Click the void button on a payment row to cancel it. The payment stays in the history as Cancelled for audit purposes.

Voiding a Final payment will not automatically revert the project stage. Contact your administrator if a manual stage correction is needed.

Warranty Tracker

Sidebar → Warranty shows all projects currently inside the warranty period.

ColumnMeaning
ProjectLinked project name
Start DateWhen the warranty began (date of final payment)
End DateOne year from start date
Days RemainingGreen = healthy · Orange = under 30 days · Red = expired

The system auto-expires records daily. You can also manually expire any record — this sets the project stage to Warranty expired immediately.

Timeline

Sidebar → Timeline shows a 3-month visual Gantt of project milestones — 14 days behind today and 76 days ahead. A red line marks today.

  • Blue dot — Installation milestone
  • Green dot — Delivery / material delivery
  • Purple dot — General project activity

Only projects with upcoming dates appear. Hover any dot for details.

Calendar

Sidebar → Calendar shows all system events. Use the tabs to filter:

  • All — every event
  • Project Activity — fabrication and activity events
  • Payments — payment received and payment due events
  • Installation — installation and delivery
  • Material Delivery — delivery events only
Payment events are only visible to you and the manager. Other roles see the calendar without financial details.

Timesheets

Sidebar → Timesheets shows all labour hours logged against projects.

Filtering

Use the date range pickers (From / To) and the worker dropdown to narrow results. The table updates live.

Editing an Entry

1Click Edit on the row you want to change.
2Update regular hours, overtime hours, and notes. Maximum 16 hours total per day.
3Save. Totals and estimated cost update automatically.

Deleting an Entry

Click the delete button on a row and confirm. This cannot be undone.

Exporting

Click Export Excel to download the currently filtered entries as a spreadsheet.

User Management

Sidebar → Users shows every system account.

Adding a User

1Click "Add User".
2Fill in: Name, Email, Password (8+ characters), Role.

Optionally enter the Airtable Member ID if this user is an SED tracked in Airtable.

3Submit. The user can log in immediately.

Deactivating a User

Click Deactivate on any row. The account is disabled — they can't log in — but all their data is preserved. You can reactivate at any time.

Accountant Email Setting

At the bottom of the Users page, set the accountant's email. This address receives automatic notifications when payments are recorded and when projects close.

Task Status Reference

Every task in the system moves through these statuses in order:

Locked → To Do → In Progress → Pending Approval → Completed
StatusWhat it means
LockedWaiting for a previous task to be completed first
To DoReady to start — all prerequisites are done
In ProgressActively being worked on
Pending ApprovalSubmitted and waiting for your review
CompletedDone — unlocks the next task in the chain

FAQ

A project isn't advancing — what's blocking it?

Go to Phase Gates and expand the project card. It lists every blocking task by name and shows its current status. Find the team member responsible for that task and ask them to complete it. Once all tasks are Completed, the Advance button will work.

How do I void a wrong payment?

Go to Payments, expand the project, and click the void button on the payment row. The status changes to Cancelled but the record stays in the history for audit purposes. Then record the correct payment using "+ Add Payment".

How do I see which SED is underperforming?

On the Dashboard Overview, look at the SED Performance Chart. A SED with many projects stuck in Preparing or Not Approved may need follow-up. Click their name pill to see the full project list filtered to that SED. The commission table below the chart shows revenue collected per SED.

How do I leave a note that all team members can see on a task?

Open and expand any task card — from Team Activity, My Tasks, or the project detail page. Scroll to the amber Admin Follow-up Note field. Type your note — it saves automatically. All roles assigned to that task will see it as a pinned amber banner. Only you can edit it.

A task shows "Pending Approval" but I wasn't notified — how do I find it?

Go to My Tasks and look in the Pending Approvals section. All tasks waiting for your review are listed there. Expand the card to review and approve or reject.

How do I expire a warranty manually?

Go to Warranty Tracker, find the record, and click the expire button. The maintenance record changes to Expired and the project stage updates to Warranty expired immediately.

How do I download a report as Excel?

On the Dashboard Overview, scroll to the Reports section. Set your quarter and year, then click the Excel icon next to the report you want. The file downloads immediately to your browser.

How do I deactivate a user account?

Go to User Management (sidebar → Users), find the user in the list, and click the Deactivate button. Their account is immediately disabled — they cannot log in. Their data and history are fully preserved. You can reactivate them at any time from the same page.
